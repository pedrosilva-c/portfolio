LAB-002 · Windows Server

File Server e Compartilhamentos NTFS

Configuração de um servidor de arquivos corporativo, com compartilhamentos de rede organizados por departamento e permissões NTFS aplicadas de forma granular.

Concluído

Objetivo

Construir um servidor de arquivos centralizado, com pastas compartilhadas organizadas por departamento e controle de acesso baseado em grupos do Active Directory.

Processo

  • Instalação da função de Serviços de Arquivo e Armazenamento no Windows Server.
  • Criação da estrutura de pastas por departamento (Financeiro, RH, TI, Diretoria).
  • Configuração de compartilhamentos de rede (SMB) para cada pasta departamental.
  • Aplicação de permissões NTFS específicas por grupo de segurança do AD.
  • Testes de acesso com diferentes usuários para validar o isolamento entre departamentos.
  • Configuração de cotas de armazenamento por pasta.

Desafios encontrados

Alinhar corretamente as permissões de compartilhamento (nível de rede) com as permissões NTFS (nível de sistema de arquivos) foi o ponto mais delicado, já que a permissão efetiva é sempre a mais restritiva entre as duas camadas.

Resultado

Servidor de arquivos funcional, com cada departamento acessando apenas suas próprias pastas, cotas de espaço aplicadas e trilha de auditoria básica configurada.
